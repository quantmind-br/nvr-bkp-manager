<objective>
## Goal
Connect to the remote Storage Box via SFTP, expose a file listing API, and render a browsable file list in the frontend — proving end-to-end connectivity from browser to remote storage.

## Purpose
This is the first half of the MVP: if the app can connect to the Storage Box and list files, the user can see their recordings. Combined with Phase 3 (streaming), this validates the core concept.

## Output
- SFTP service module with connection pooling
- GET /api/files endpoint returning file metadata (name, size, date, type)
- React file browser component with directory navigation
</objective>

## AC-2: File listing API returns metadata
```gherkin
Given the backend is connected to the Storage Box
When GET /api/files is called (optionally with ?path= query param)
Then it returns a JSON array of file objects with: name, size, modifiedAt, isDirectory
```

```gherkin
Given the Storage Box has nested directories
When GET /api/files?path=/subdir is called
Then it returns files within that subdirectory
And includes a ".." entry for parent navigation (unless at root)
```

## AC-4: Frontend displays file list
```gherkin
Given the backend returns file metadata
When the user opens the app in the browser
Then they see a table/list of files with name, size, date, and type icon
And can click directories to navigate into them
```

<task type="auto">
  <name>Task 1: Create SFTP service with connection management</name>
  <files>backend/src/services/sftp.ts, backend/package.json</files>
  <action>
    Install ssh2-sftp-client package:
    - npm install ssh2-sftp-client
    - npm install -D @types/ssh2-sftp-client

    Create backend/src/services/sftp.ts:
    - Import ssh2-sftp-client and config
    - Export async function listFiles(remotePath?: string) that:
      1. Creates a new SftpClient instance
      2. Connects using config.storage (host, port, user, password)
      3. Lists files at config.storage.path + remotePath (default "/")
      4. Maps each entry to: { name, size, modifiedAt (ISO string), isDirectory }
      5. Disconnects after listing
      6. Returns the mapped array
    - Export async function testConnection() that connects, lists root, disconnects — returns true/false
    - Handle connection errors gracefully (throw typed errors with meaningful messages)
    - Do NOT use connection pooling yet (add in future optimization) — keep it simple: connect, list, disconnect

    Avoid: Leaving connections open (always disconnect in finally block)
    Avoid: Exposing raw ssh2-sftp-client types outside the service
  </action>
  <verify>
    Create a quick test: add a temporary GET /api/storage/test route that calls testConnection() and returns the result.
    Run backend with real .env credentials, curl /api/storage/test → should return { connected: true }.
    Then remove the temporary test route.
  </verify>
  <done>AC-1 satisfied: SFTP service connects to Storage Box and lists files</done>
</task>

<task type="auto">
  <name>Task 2: Create file listing API route</name>
  <files>backend/src/routes/files.ts, backend/src/index.ts</files>
  <action>
    Create backend/src/routes/files.ts:
    - Export async function fileRoutes(app: FastifyInstance)

    - Register with app.register(fileRoutes)
  </action>

  </verify>
