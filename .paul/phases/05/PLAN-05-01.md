# PLAN-05-01: Authentication & Security

## Phase
Phase 5: Authentication & Security

## Objective
Add JWT authentication, role-based access control (admin/viewer), SQLite user persistence, and route protection to all API endpoints — achieving LGPD compliance (zero public routes).

## Steps

### Step 1: Database + User Service
- Install: better-sqlite3, bcryptjs, @fastify/jwt, fastify-plugin + types
- Create backend/src/db.ts (SQLite singleton, WAL mode)
- Create backend/src/services/users.ts (CRUD, bcryptjs hash/verify)
- Create backend/src/seed.ts (init table, seed admin if empty)
- Update config.ts with jwt.secret, jwt.expiresIn, defaultAdminPassword
- Update .env/.env.example with JWT_SECRET, DEFAULT_ADMIN_PASSWORD

### Step 2: JWT Plugin + Auth Routes + Route Protection
- Create backend/src/plugins/auth.ts (fastify-plugin, onRequest hook, requireRole)
- Create backend/src/routes/auth.ts (POST /api/auth/login, GET /api/auth/me)
- Update index.ts (seedDatabase, register authPlugin before routes)
- Update files.ts (requireRole("admin") on DELETE and POST)
- Update docker-compose.yml (backend-data volume for SQLite)

### Step 3: Frontend Auth
- Create frontend/src/auth.tsx (AuthContext, AuthProvider, useAuth)
- Create frontend/src/api.ts (apiFetch with auto Authorization header)
- Create frontend/src/components/LoginPage.tsx
- Update main.tsx (wrap in AuthProvider)
- Update App.tsx (auth gate, logout button, user info)
- Update FileList.tsx (apiFetch, hide admin-only UI for viewers)
- Update UploadButton.tsx (apiFetch)
- Update VideoPlayer.tsx (append token to stream URL)

## Acceptance Criteria
- [ ] Unauthenticated requests to /api/files return 401
- [ ] POST /api/auth/login with valid credentials returns JWT
- [ ] JWT-authenticated requests to /api/files return 200
- [ ] Viewer role cannot DELETE or POST upload (403)
- [ ] Admin role can perform all operations
- [ ] Frontend shows login page when not authenticated
- [ ] Frontend shows file list after successful login
- [ ] Video streaming works with token query param
- [ ] Default admin user seeded on first startup
- [ ] Docker Compose works with SQLite volume persistence
