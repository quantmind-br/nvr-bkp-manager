## Goal
Add download, upload, and delete operations for files on the remote Storage Box, all accessible through the web interface.

## Purpose
Completes the CRUD cycle: users can now browse (Phase 2), watch (Phase 3), and manage (Phase 4) their recordings entirely through the browser — eliminating the need for FTP clients.

## Output
- SFTP delete and upload functions in sftp.ts
- GET /api/download, DELETE /api/files/:name, POST /api/upload endpoints
- Download and Delete buttons in FileList
- Upload button component
</objective>

## AC-1: Download files via browser

```gherkin
Given a file exists on the Storage Box
When the user clicks "Delete" and confirms the action

```

## AC-3: Upload files to Storage Box
```gherkin
Given the user is viewing the file list
When they click "Upload" and select one or more files

And the file list refreshes to show the new files
And upload progress is visible
```

```gherkin

```

<task type="auto">
  <name>Task 1: Add SFTP operations and backend API routes</name>
  <files>backend/src/services/sftp.ts, backend/src/routes/files.ts, backend/src/index.ts</files>
  <action>
    **Extend sftp.ts with new functions:**

    1. `deleteFile(fileName: string)`:
       - Connect, resolve full path via normalizePath
       - Call sftp.delete(fullPath)
       - Disconnect in finally block
       - Throw on error with meaningful message

    2. `uploadFile(fileName: string, dataStream: Readable)`:
       - Connect, resolve full path via normalizePath
       - Call sftp.put(dataStream, fullPath)
       - Disconnect in finally block
       - Throw on error

    3. `getFileSize(fileName: string): Promise<number>`:
       - Connect, stat the file, return size
       - Needed for Content-Length on downloads
       - Disconnect in finally block

    **Extend files.ts with new routes:**

    1. `GET /api/download?file=<name>`:
       - Validate filename (no path separators or "..")
       - Get file size via getFileSize
       - Get read stream via getReadStream
       - Set headers: Content-Type application/octet-stream, Content-Disposition attachment with filename, Content-Length
       - Pipe stream to reply.raw (same pattern as stream.ts)
       - Cleanup SFTP on end/error/close
       - Use reply.hijack()

    2. `DELETE /api/files?file=<name>`:
       - Validate filename
       - Call deleteFile(fileName)
       - Return { success: true, deleted: fileName }
       - On error: 502 with { error, details }

    3. `POST /api/upload`:
       - Use @fastify/multipart for file upload handling
       - Install: npm install @fastify/multipart
       - Register multipart plugin in index.ts with fileSize limit (2GB)
       - Accept multipart form data with file field
       - For each file part: call uploadFile(originalFilename, fileStream)
       - Return { success: true, uploaded: [filenames] }
       - On error: 502 with error details

    **Register @fastify/multipart in index.ts** (before routes, with limits config)
  </action>
  <verify>
    Start backend with npm run dev:
    - curl download: curl -o /tmp/test-dl.dav "http://localhost:3001/api/download?file=<small-dav>" → file downloaded with correct size
    - curl upload: curl -F "file=@/tmp/test-dl.dav" "http://localhost:3001/api/upload" → uploaded successfully
    - curl delete: curl -X DELETE "http://localhost:3001/api/files?file=<uploaded-file>" → deleted
    - Verify file is gone from Storage Box
  </verify>
