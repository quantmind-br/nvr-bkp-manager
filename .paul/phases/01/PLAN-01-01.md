# PLAN-01-01: Project Foundation

## Phase

Phase 1: Project Foundation

## Objective

Scaffold the full project structure with a working Docker Compose dev environment. Establish the monorepo layout, configure TypeScript across frontend and backend, set up the build toolchain, and verify that all containers start and communicate correctly.

## Context

This is the first plan вЂ” no existing code. The stack is decided:
- Frontend: React + Vite (TypeScript)
- Backend: Node.js + Fastify (TypeScript)
- Database: SQLite (via better-sqlite3 or drizzle-orm)
- Transcoding: FFmpeg (available in backend container)
- Storage protocol: SFTP
- Orchestration: Docker Compose

Fastify chosen over Express for performance (faster request handling, native JSON serialization, built-in schema validation).

### Step 2: Configure frontend (React + Vite + TypeScript)
- Initialize React + Vite project with TypeScript template
- Configure vite.config.ts with API proxy to backend (dev mode)
- Create minimal App.tsx that renders a "nvr-bkp-manager" heading
- Verify `npm run dev` starts without errors

**Acceptance:** `npm run dev` in frontend/ serves the app on localhost with no errors.

### Step 3: Configure backend (Fastify + TypeScript)
- Initialize Node.js project with TypeScript
- Install fastify, @fastify/cors, dotenv
- Create server entry point with GET /api/health returning `{ status: "ok", timestamp: <iso> }`
- Load config from environment variables via .env
- Verify `npm run dev` starts without errors

**Acceptance:** `curl localhost:3001/api/health` returns 200 with JSON payload.

### Step 4: Docker Compose setup
- Create Dockerfile.frontend (multi-stage: build + nginx serve)
- Create Dockerfile.backend (node:20-slim + FFmpeg installed via apt)
- Create docker-compose.yml with services: frontend, backend
- Create docker-compose.dev.yml with volume mounts and hot reload
- Configure nginx.conf to proxy /api/* to backend, serve frontend for everything else
- Verify FFmpeg is available inside backend container: `ffmpeg -version`

**Acceptance:** `docker compose up --build` starts both services. Frontend accessible on port 80, API proxied correctly. `docker compose exec backend ffmpeg -version` succeeds.

### Step 5: Git initialization and .env setup
- Initialize git repo
- Create comprehensive .gitignore (node_modules, dist, .env, *.db, .DS_Store)
