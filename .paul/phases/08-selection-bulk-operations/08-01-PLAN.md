<objective>
## Goal
Add multi-select checkboxes to the file list, a selection status bar, bulk delete with confirmation and partial failure handling, and bulk download via server-side zip streaming.

## Purpose
Users managing NVR backups need to act on multiple files at once — deleting old recordings across different days or downloading a batch for offline review. Single-file operations are too slow when dealing with dozens or hundreds of files.

## Output
- New `archiver` dependency in backend
- New `POST /api/bulk-delete` endpoint (accepts file list, returns per-file results)
- New `POST /api/bulk-download` endpoint (streams zip archive of requested files)
- Modified `frontend/src/components/FileList.tsx` — checkboxes, select all, selection bar, bulk action buttons
</objective>

<task type="auto">
  <name>Task 1: Backend bulk endpoints — delete and zip download</name>
  <files>backend/src/routes/files.ts, backend/package.json</files>
  <action>
    1. **Install archiver:**
       ```bash
       cd backend && npm install archiver && npm install -D @types/archiver
       ```

    2. **Add import** at top of files.ts:
       ```typescript
       import archiver from "archiver";
       ```

    3. **POST /api/bulk-delete** (admin only):
       - Accept JSON body: `{ files: string[] }`
       - Validate: non-empty array, each filename passes validateFileName()
       - Iterate files, attempt deleteFile() on each
       - Collect results: `{ file: string, success: boolean, error?: string }[]`
       - Log each successful deletion via logAction()
       - Return `{ results }` with 200 (even if some fail — partial success is OK)
       - Prehandler: requireRole("admin")

    4. **POST /api/bulk-download:**
       - Accept JSON body: `{ files: string[] }`
       - Validate: non-empty array, max 50 files, each passes validateFileName()
       - Create archiver("zip", { zlib: { level: 1 } }) — level 1 for speed since video files don't compress much
       - For each file:
         - Call getReadStream(fileName)
         - archive.append(stream, { name: fileName })
         - On stream end/error: sftp.end()
       - Set response headers:
         - Content-Type: application/zip
         - Content-Disposition: attachment; filename="nvr-recordings.zip"
       - Pipe archive to reply.raw
       - Call archive.finalize()
       - Return reply.hijack()
       - Handle archive errors: destroy streams, close SFTP connections

    5. **Interface for bulk request body:**
       ```typescript
       interface BulkFilesBody { files: string[] }
       ```
  </action>

    Check archiver is in package.json dependencies
  </verify>
