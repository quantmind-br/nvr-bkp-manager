# PLAN-06-01: Audit & Production Deploy

## Phase
Phase 6: Audit & Production Deploy

## Objective
Add activity logging for critical actions (LGPD compliance), push to a Git remote, and deploy the application via Dokploy to https://nvr.quantmind.com.br.

## Context
- All features built (Phases 1-5): browse, stream, CRUD, auth
- SQLite DB exists with users table — add audit_logs table
- Dokploy at $DOKPLOY_URL with $DOKPLOY_API_KEY
- Production URL: https://nvr.quantmind.com.br
- No git remote yet — need to create GitHub repo and push
- Use the `dokploy-management` skill for deploy operations

## Steps

### Step 1: Audit logging service and middleware
- Create `backend/src/services/audit.ts`:
  - initAuditTable() — create audit_logs table (id, user_id, username, action, resource, details, ip, created_at)
  - logAction(userId, username, action, resource, details?, ip?) — insert log entry
- Create `backend/src/routes/audit.ts`:
  - GET /api/audit — return recent audit logs (admin only, paginated with ?limit=&offset=)
- Update `backend/src/seed.ts` — call initAuditTable()
- Add audit logging calls to:
  - `backend/src/routes/auth.ts` — log login success/failure
  - `backend/src/routes/files.ts` — log delete, upload
  - `backend/src/routes/stream.ts` — log video stream (file name)

### Step 2: Push to GitHub remote
- Create GitHub repo (private) via `gh` CLI
- Add remote, push all commits
- Verify repo accessible

### Step 3: Deploy via Dokploy
- Use `dokploy-management` skill to:
  - Create a Compose project in Dokploy
  - Configure it with the GitHub repo
  - Set environment variables (storage, JWT, admin password)
  - Trigger deployment
  - Verify at https://nvr.quantmind.com.br

## Acceptance Criteria
- [ ] Audit logs table created on startup
- [ ] Login/delete/upload/stream actions logged with user, action, resource, timestamp
- [ ] GET /api/audit returns logs (admin only)
- [ ] Code pushed to GitHub remote
- [ ] Application deployed and running via Dokploy
- [ ] https://nvr.quantmind.com.br shows login page
- [ ] Full flow works in production: login → browse → play → download
