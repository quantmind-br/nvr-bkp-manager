<objective>
## Goal
Upgrade the file list filtering system: replace single-channel dropdown with multi-channel checkboxes, add time-of-day range filter, add file size range filter, and improve filter controls layout with visual grouping.

## Purpose
Current filtering is basic (one channel, one date). Users managing large volumes of NVR recordings need to filter by multiple channels simultaneously, narrow by time-of-day (e.g., "show only nighttime recordings"), and filter by file size — enabling efficient bulk operations in Phase 8.

## Output
- Modified `backend/src/routes/files.ts` — accept `minSize`/`maxSize` query params, accept comma-separated `channel` param
- Modified `frontend/src/components/FileList.tsx` — new filter UI with multi-channel, time-of-day range, size range, and grouped layout
</objective>

<acceptance_criteria>

## AC-1: Multi-channel filtering
```gherkin
Given the file list contains recordings from channels "ch01", "ch02", "ch03"
When the user selects "ch01" and "ch03" in the channel filter
Then only recordings from ch01 and ch03 are displayed (plus directories)
```

## AC-2: Time-of-day range filtering
```gherkin
Given recordings exist with start times across the full day
When the user sets time-of-day filter from "22:00" to "06:00" (nighttime)
Then only recordings whose start time falls within 22:00-23:59 or 00:00-05:59 are shown
```

## AC-3: File size range filtering
```gherkin
Given recordings range from 10MB to 2GB
When the user sets minimum size to 100MB
Then only files >= 100MB are displayed
And the size filter accepts both min and max values independently
```

## AC-4: Filter controls are visually grouped
```gherkin
Given the filter bar is visible
When the user views the filter controls
Then filters are organized in labeled groups (Channel, Date, Time, Size)
And each group has a visible border or background separation
And the layout wraps cleanly on narrow screens
```

  <name>Task 1: Extend backend filtering — multi-channel and size range</name>
  <files>backend/src/routes/files.ts</files>
  <action>
    1. Update FileListQuery interface to add `minSize` and `maxSize` optional string params
    2. Update FileFilters interface to add `minSize?: number` and `maxSize?: number`
    3. In the GET /api/files handler, parse `minSize` and `maxSize` from query (parse as integers, ignore if NaN or negative)
    4. Update `channel` query param to support comma-separated values (e.g., "ch01,ch03"). Split on comma, trim, lowercase each. Store as `string[]` in FileFilters
    5. Update `shouldIncludeFile()`:
       - Channel filter: check if file's channel matches ANY of the channels in the array (instead of single match)
       - Size filter: if minSize set, exclude files where `file.size < minSize`. If maxSize set, exclude files where `file.size > maxSize`
       - Directories always pass all filters (existing behavior)
    6. Validate: minSize/maxSize must be non-negative integers if provided, return 400 on invalid values
  </action>
  <verify>
    Start backend dev server. Test with curl:
    - `curl /api/files?channel=ch01,ch02` returns files from both channels
    - `curl /api/files?minSize=1048576` returns only files >= 1MB
    - `curl /api/files?minSize=100&maxSize=999999` returns files in range
    - Existing date filters still work unchanged
  </verify>
  <done>AC-1 backend: multi-channel param works. AC-3 backend: size filtering works.</done>
