<objective>
## Goal
Stream .dav and .mp4 video files from the Storage Box to the browser in real-time, with FFmpeg remuxing .dav (raw H.265/HEVC) into fragmented MP4 on-the-fly.

## Purpose
This is the core technical proof — the MVP differentiator. If .dav files play in the browser, the project's core value is validated: users no longer need proprietary DVR software.

## Output
- SFTP read stream function for remote files
- FFmpeg streaming service (SFTP → FFmpeg remux → HTTP response)
- GET /api/stream endpoint
- Frontend VideoPlayer component with HTML5 video tag
- Play button integrated into FileList

## Technical Discovery (Pre-validated)
Research during planning confirmed:
- .dav files contain **raw H.265/HEVC** NAL units with a Dahua proprietary header
- FFmpeg reads them with `-f hevc` input format
- **Remux only** (`-c:v copy`) — no transcoding, minimal CPU usage
- Pipe streaming works: `SFTP stream → FFmpeg stdin → FFmpeg stdout → HTTP response`
- Output: fragmented MP4 (`-movflags frag_keyframe+empty_moov+default_base_moof`) plays natively in HTML5 `<video>`
- .mp4 files can be proxied directly from SFTP without FFmpeg
</objective>

<task type="auto">
  <name>Task 1: Add SFTP read stream and streaming service</name>
  <files>backend/src/services/sftp.ts, backend/src/services/stream.ts</files>
  <action>
    **Extend sftp.ts:**
    - Add function `getReadStream(remotePath: string)` that:
      1. Creates new SftpClient, connects
      2. Resolves full path via normalizePath
      3. Creates a read stream via sftp.createReadStream(fullPath)
      4. Returns { stream, sftp } (caller manages lifecycle — sftp.end() after stream ends)

    **Create backend/src/services/stream.ts:**
    - Import child_process spawn and getReadStream
    - Export function `createVideoStream(fileName: string)`:
      1. Call getReadStream with the file path
      2. Determine file extension
      3. If .dav:
         - Spawn FFmpeg: `ffmpeg -f hevc -i pipe:0 -c:v copy -f mp4 -movflags frag_keyframe+empty_moov+default_base_moof pipe:1`
         - Pipe SFTP read stream into FFmpeg stdin
         - Return { stream: ffmpeg.stdout, cleanup: function }
         - cleanup() must: kill FFmpeg process, end SFTP connection
         - Handle FFmpeg stderr for error detection (log errors, don't pipe to response)
         - Handle SFTP stream errors (destroy FFmpeg process on error)
      4. If .mp4:
         - Return { stream: sftpReadStream, cleanup: function }
         - cleanup() must: end SFTP connection
      5. If unsupported extension: throw error

    **Error handling:**
    - If SFTP connection fails, throw with meaningful message
    - If FFmpeg process exits with error, log stderr and destroy output stream
    - Always clean up resources (SFTP connection, FFmpeg process) on stream end or error

    Avoid: Buffering entire file in memory — everything must stream
    Avoid: Leaving zombie FFmpeg processes — always kill on cleanup
  </action>
  <verify>
    Quick integration test: start backend, curl /api/stream?file=<dav-file> > /tmp/test.mp4
    Verify output is valid MP4: ffprobe /tmp/test.mp4 shows video stream
  </verify>

<task type="auto">
  <name>Task 2: Create streaming API endpoint</name>
  <files>backend/src/routes/stream.ts, backend/src/index.ts</files>
  <action>
    **Create backend/src/routes/stream.ts:**
    - Export async function streamRoutes(app: FastifyInstance)
    - Register GET /api/stream with required query param: file (string)
    - Validate: file must not contain path separators or ".." (security — only filenames, not paths)
    - For now, files are in the root storage path (no subdirectory support for streaming yet)
    - Call createVideoStream(file)
    - Set response headers:
      - Content-Type: video/mp4
      - Transfer-Encoding: chunked (implicit with streaming)
      - Cache-Control: no-cache
    - Use reply.raw to pipe the stream directly to the HTTP response
    - On stream end: call cleanup()
    - On stream error: call cleanup(), send 502 if headers not yet sent
    - On client disconnect (request.raw 'close' event): call cleanup() to stop FFmpeg and SFTP

    **Register in backend/src/index.ts:**
    - Import streamRoutes
    - Register with app.register(streamRoutes)

    **Important:** Use reply.raw (Node.js raw response) for streaming, not Fastify's reply.send(),
    because Fastify's reply.send() buffers the response. Set reply.raw headers manually and pipe directly.
  </action>
